# About Chrome Extension Guide

I built my first Chrome extension in 2019 and quickly realized there was no single place with comprehensive, up-to-date documentation for extension developers. The official docs are good but sparse. Stack Overflow answers are often outdated. Blog posts rarely cover edge cases.

Chrome Extension Guide fills that gap. With 700+ articles covering APIs, permissions, design patterns, tutorials, and publishing, this is the most thorough independent resource for Chrome extension development on the web.

## What we cover

- **Guides** -- in-depth walkthroughs of extension architecture, service workers, content scripts, and message passing
- **Design patterns** -- authentication flows, state management, storage encryption, and performance profiling
- **API reference** -- complete documentation for Tabs, Windows, Bookmarks, History, Downloads, Alarms, Notifications, and more
- **Manifest V3** -- migration strategies, service worker patterns, promise-based APIs, and offscreen documents
- **Tutorials** -- hands-on projects like building a bookmark manager, dark mode toggle, color picker, and AI writing assistant
- **Publishing** -- Chrome Web Store submission, listing optimization, beta testing, and common rejection reasons
- **Permissions** -- detailed documentation on every Chrome permission with usage examples and security considerations

## Who this is for

Developers building Chrome extensions, from beginners creating their first popup to experienced engineers migrating complex extensions to Manifest V3. If you write JavaScript or TypeScript for the Chrome platform, this site is for you.
