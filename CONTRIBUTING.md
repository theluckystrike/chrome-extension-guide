# CONTRIBUTING

Thank you for your interest in contributing to the Chrome Extension Guide.

## REPORTING ISSUES

Found a typo in the documentation or something that could be improved? Please open an issue on GitHub. Include as much detail as possible, including:

- The page where you found the issue
- A clear description of the problem or suggestion
- Any relevant code snippets or links

## DEVELOPMENT WORKFLOW

1. Fork the repository
2. Clone your fork
3. Create a new branch for your changes
4. Make your edits
5. Submit a pull request

## CODE STYLE

- Use clear, descriptive headings
- Keep code examples concise and well-commented
- Follow TypeScript best practices
- Ensure all links are valid

## TESTING

This is a documentation project. When adding new guides or API references:

- Verify all code examples compile correctly
- Check that all internal links work
- Ensure external resources are accessible
